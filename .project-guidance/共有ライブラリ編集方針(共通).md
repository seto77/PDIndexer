# 共有ライブラリ編集方針（共通）

- 作成: 2026-06-01 (260601Ch)
- 適用範囲: ReciPro / PDIndexer / CSManager / IPAnalyzer
- 対象: `Crystallography`, `Crystallography.Controls`, `Crystallography.Native`
- 目的: 4 アプリで共有しているライブラリを安全に変更する。

> このファイルは 4 リポジトリ共通方針です。特定アプリだけの不具合調査や適用ログは各アプリ固有ファイルへ書いてください。

---

## 1. 共有リポジトリの役割

- `Crystallography`: 結晶学・数値計算・共通モデルのコアライブラリ。
- `Crystallography.Controls`: WinForms 共通コントロール、フォーム、キャプチャ補助など。
- `Crystallography.Native`: C++/Eigen を使うネイティブライブラリ。ビルドは Visual Studio で確認する。

各アプリの作業ツリーには共有ライブラリが含まれているが、実質的に横断依存している。1 アプリで直した共有側変更は、他 3 アプリにも影響する可能性が高い。

## 2. 変更判断

- アプリ固有の挙動は、まずアプリ側で解決できるか検討する。
- 4 アプリに共通する UI 部品、数値処理、描画補助、ローカライズ補助は共有側に置く。
- 共有側へアプリ固有の条件分岐を入れる場合は、他アプリで意味を持つか、名前と責務が自然かを確認する。
- 共有側の public API を変えるときは、既存呼び出しを検索し、後方互換を優先する。
- 新しいパラメータは、可能ならオーバーロードではなくデフォルト引数で対応する。

## 3. WinForms 共有コントロール

- Designer で使われる public property、constructor、event は安易に削除・リネームしない。
- `[Browsable]`, `[DesignerSerializationVisibility]`, `[DefaultValue]` などの design-time 属性は Designer 出力に影響する。
- `components`、子コントロール、Dispose の所有関係を確認する。
- tooltip は host フォーム側 provider を基本とし、内部子への relay が必要な場合は共有側で扱う。詳細は `ツールチップ編集方針(共通).md` を参照する。
- 高 DPI、フォント変更、日本語表示で文字切れや位置ずれがないか確認する。

## 4. 計算・データ処理

- 結晶学的な式、定数、単位変換を変更する場合は、影響範囲をコメントまたは個別メモに残す。
- 既存の数値結果が変わる変更は、バグ修正なのか仕様変更なのかを明記する。
- キャッシュ、静的テーブル、外部データの生成物を更新した場合は、再生成手順を残す。
- アプリ固有の都合で共有計算を変えない。必要ならオプションや呼び出し側の前処理で分ける。

## 5. Native

- `Crystallography.Native` は Visual Studio でビルド確認する。
- C# wrapper を変えた場合は、DLL の配置、platform target、例外時の挙動を確認する。
- Windows on ARM、OpenGL、GPU/driver 依存の問題は、まず再現条件を個別調査メモへ残し、4 アプリへ効く結論だけ共通方針へ要約する。

## 6. 検証

共有側を変更した場合の基本確認:

- [ ] 変更元アプリで `dotnet build`。
- [ ] 可能なら他 3 アプリでも `dotnet build`。
- [ ] WinForms 共有コントロールなら、少なくとも代表フォームで表示確認。
- [ ] resx や Designer 差分が意図したものか確認。
- [ ] tooltip、Pages、ローカライズに横断知見が出た場合は該当共通方針を更新。
