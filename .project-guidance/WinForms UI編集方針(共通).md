# WinForms UI 編集方針（共通）

- 作成: 2026-06-01 (260601Ch)
- 適用範囲: ReciPro / PDIndexer / CSManager / IPAnalyzer
- 目的: 4 アプリの WinForms 画面を、同じ思想・同じ操作感で保守する。

> このファイルは 4 リポジトリ共通方針です。フォーム別の改善案、未対応リスト、スクリーンショット監査は各アプリ固有ファイルへ書いてください。

---

## 1. 基本方針

- UI は「既存アプリ群の操作感を揃える」ことを優先する。
- 新規 UI は、既存のフォーム、メニュー、共有コントロールのパターンに合わせる。
- 目立つ再設計より、科学ソフトとしての密度、読みやすさ、予測可能性を優先する。
- GUI 文言、tooltip、Pages マニュアルは同じ用語を使う。用語の詳細は `ローカライズ・用語方針(共通).md` を参照する。

## 2. resx と Designer

- 表示文字列は原則として resx に置く。コードへ直接文字列を埋め込まない。
- 英語は既定 `*.resx`、日本語は `*.ja.resx` を使う。
- Designer 生成ファイルは必要最小限に触る。Visual Studio Designer で保存した後は、意図しない位置・サイズ・resx 改行の差分を確認する。
- 手編集が必要な場合も、Designer が再生成する領域と通常コードを混ぜない。
- `components` を追加したフォームでは `Dispose` で破棄されることを確認する。

## 3. レイアウト

- 既存フォームの密度に合わせ、余白を増やしすぎない。
- 単位付き入力は、値、単位、説明の位置関係を崩さない。
- ラベルは対象コントロールの近くに置き、同じ種類の入力は列を揃える。
- メニュー、ツールバー、ステータスバーは既存アプリの順序・名前・振る舞いに合わせる。
- 画面端、DPI 150/200%、日本語表示で文字切れがないか確認する。
- 自動サイズ調整で Designer 差分が大きくなる場合は、影響範囲を明記する。

## 4. 共有コントロール

- `Crystallography.Controls` に既存の部品がある場合は、アプリ固有の新規部品を作る前に再利用できるか確認する。
- 共有コントロールを変更する場合は、4 アプリへの波及を想定する。詳細は `共有ライブラリ編集方針(共通).md` を参照する。
- 共有コントロールのデザイナ公開プロパティを増やすと、既存フォームの Designer 生成差分やシリアライズに影響する。追加時は慎重に行う。
- tooltip は host フォーム側 provider を基本にする。詳細は `ツールチップ編集方針(共通).md` を参照する。

## 5. 入力・操作

- ボタンは「即時実行」、チェックボックスは「状態」、コンボボックスは「選択肢」、数値入力は「単位と範囲」が分かるようにする。
- 1 つの操作に複数の入口を作る場合は、メニュー、ショートカット、ボタンの文言と有効/無効状態を揃える。
- エラーや警告は、ユーザーが次に何を直せばよいか分かる文にする。
- 既存の keyboard shortcut や menu mnemonic を壊さない。

## 6. 変更時チェック

- [ ] `dotnet build` が通る。
- [ ] EN/JA の resx が揃っている。
- [ ] 日本語 UI で文字切れがない。
- [ ] 高 DPI で主要フォームが破綻しない。
- [ ] tooltip と Pages マニュアルに影響があれば該当方針を更新する。
- [ ] 共有コントロールを触った場合は、他アプリの影響を確認または明記する。
