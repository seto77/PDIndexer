# ローカライズ・用語方針（共通）

- 作成: 2026-06-01 (260601Ch)
- 適用範囲: ReciPro / PDIndexer / CSManager / IPAnalyzer
- 目的: GUI、tooltip、Pages マニュアルで EN/JA の内容と専門用語を揃える。

> このファイルは 4 リポジトリ共通方針です。アプリ固有の用語辞書や未翻訳リストは各アプリ固有ファイルへ書いてください。

---

## 1. 正本

- GUI 表示文字列は resx を正とする。
- 英語 UI は既定 `*.resx`、日本語 UI は `*.ja.resx` を正とする。
- Pages や tooltip で UI 要素名を書くときは、翻訳で推測せず、対象言語の現行 resx ラベルに合わせる。
- 旧マニュアル、過去のスクリーンショット、別アプリ由来の文言より、現在の GUI 表示を優先する。

## 2. EN/JA 同期

- 手書きコンテンツのリード言語は原則として日本語とする。
- 日本語を編集したら英語にも同じ意味を反映する。
- 英語だけが先行している場合は、日本語への反映を提案する。
- 同時編集で内容が食い違う場合は、勝手に混ぜず、ページまたは UI 要素単位で正を決める。
- 一方の言語にだけ tooltip や説明文がある状態を見つけたら、もう一方へ反映するか、未対応として個別監査ファイルへ記録する。

## 3. UI 要素名

- ボタン、メニュー、タブ、グループボックス、列見出し、チェックボックスは、本文中でも GUI と同じ表記にする。
- 英語本文では英語 resx のラベル、日本語本文では日本語 resx のラベルを使う。
- UI 要素名は、文章として自然になるよう必要最小限の助詞を付けるが、ラベル本体は変えない。
- ラベル変更をした場合は、tooltip、Pages、スクリーンショット説明も影響確認する。

## 4. 科学用語と単位

- 結晶学・回折・電子顕微鏡などの専門用語は、既存アプリ群で使っている表記を優先する。
- 数式記号、ミラー指数、方位、空間群記号、元素記号は勝手に翻訳しない。
- 単位記号は原則として半角英数字・記号を使う。例: `Å`, `Å³`, `K`, `GPa`, `nm`, `keV`, `°`
- 数値と単位の分割を避ける。tooltip の折返しでも `300 K` や `1.5406 Å` の途中で改行しない。
- 英語の scientific term を日本語化すると意味が狭くなる場合は、無理に訳さず英語表記を残す。

## 5. resx 編集

- 新しい UI 文言を追加するときは、EN/JA の両方を用意する。
- 既存 key を削除・リネームすると Designer やコード参照が壊れるため、参照検索してから行う。
- Designer が resx を再保存した場合は、tooltip の手動改行や日本語文字列が壊れていないか確認する。
- 自動生成や機械置換を行った場合は、文字化け、エスケープ、改行コードを確認する。

## 6. Pages / tooltip との関係

- Pages の本文ルールは `github pages編集方針(共通).md` を参照する。
- tooltip の文案・折返し・表示時間は `ツールチップ編集方針(共通).md` を参照する。
- GUI の見た目やフォーム設計は `WinForms UI編集方針(共通).md` を参照する。
