# ツールチップ編集方針（共通）

- 作成: 2026-06-01 (260601Ch)

- 適用範囲: ReciPro / PDIndexer / CSManager / IPAnalyzer
- 目的: WinForms `System.Windows.Forms.ToolTip` の非自明な仕様と、4 アプリ共通の tooltip 文案・配線方針をまとめる。

> このファイルは 4 リポジトリ共通方針です。フォーム別の文案、配線漏れ、監査結果は各 `<App>_ツールチップ監査*.md` に書いてください。

## 1. 基本方針

- 新規 tooltip は host フォーム側の標準 `ToolTip.SetToolTip(control, text)` に寄せる。
- 長文 tooltip は、可読性、表示時間、改行位置を優先する。
- tooltip 文は EN/JA の両方を整備する。GUI 表記は対象言語の resx ラベルに合わせる。
- 共有コントロールの仕様や WinForms の罠はこの共通ファイルへ、アプリ固有の対応状況は個別監査ファイルへ書く。

## 2. バルーン表示 (`IsBalloon`)

- `IsBalloon = true` のとき、`ToolTip.BackColor` / `ForeColor` / カスタムフォントは owner-draw でない限り無視される。
- `IsBalloon` は `OwnerDraw` より優先される。両方 true にしても、バルーン描画になり owner-draw は無視される。
- 標準 `ToolTip` では「バルーン形状」と「自前描画・色変更」は両立しない。
- バルーン採用時は、誤解防止のため色やフォント指定をコードに書かない。
- 色や自前描画が必須なら `IsBalloon = false` + `OwnerDraw = true` を検討する。ただし角丸バルーンではなく矩形表示になる。

参考: Microsoft Learn `ToolTip.BackColor` / `ToolTip.OwnerDraw`

## 3. 遅延設定

長文 tooltip では表示時間を延ばしたいが、`AutoPopDelay` には罠がある。

- Windows 10 では `AutoPopDelay` が最大 5000ms にクランプされる。Windows 11 ではこのクランプはない。
- `AutoPopDelay = 5000` ちょうどは Windows 11 24H2 で tooltip が消えず無限表示になる既知バグがある。
- `AutomaticDelay` を設定すると、`AutoPopDelay` / `InitialDelay` / `ReshowDelay` が自動再計算され、個別設定が上書きされる。
- `AutomaticDelay` は触らず、3 値を個別設定する。

4 アプリ共通の標準値:

```csharp
toolTip.IsBalloon = true;
toolTip.AutoPopDelay = 10000; // 260601Ch: 5000 ちょうどは禁止。Win10 は実効 5000ms
toolTip.InitialDelay = 500;
toolTip.ReshowDelay = 100;
```

Win10 で 5 秒以上の表示が必要な特殊箇所では、ホバー型 `SetToolTip` ではなく `ToolTip.Show(text, win, duration)` の採用を検討する。

参考: Microsoft Learn `ToolTip.AutoPopDelay` / `ToolTip.AutomaticDelay`、dotnet/winforms issue #13800

## 4. 折返し

- 標準 WinForms `ToolTip` には公開された自動 word wrap 幅指定がない。
- ネイティブ tooltip は `TTM_SETMAXTIPWIDTH` で折返し可能だが、WinForms はこれを直接公開していない。
- そのため、長文 tooltip は resx 文に手で `\r\n` を入れる方式を基本にする。

折返し規約:

- 1 行は目標 55 カラム、60 カラム超で折返しを検討する。
- 全角は 2、半角は 1 として数える。
- 英単語、数値+単位、カタカナ語、漢字複合語、括弧内の短い語は分割しない。
- EN/JA は独立に折返しを調整する。
- 既に意図的な CRLF を含む値は機械処理で壊さない。
- Visual Studio Designer でフォームを保存すると resx の改行が変わる場合があるため、Designer 編集後に diff を確認する。

参考: Microsoft Learn "Implement Multiline Tooltips"

## 5. アプリ全体 ON/OFF

- `ToolTip.Active` はインスタンス単位であり、アプリ全体の全 tooltip を自動で制御しない。
- フォームやユーザーコントロールごとに別々の `ToolTip` を持つ設計では、1 つの menu handler で 1 インスタンスだけ切り替えても他の tooltip は残る。
- アプリ全体 ON/OFF が必要なら、全 `ToolTip` インスタンスを列挙して同期するか、静的レジストリで一括管理する。
- メニュー文言が「アプリ全体」を示す場合は、実装も全インスタンスに効くよう確認する。

参考: Microsoft Learn `ToolTip.Active`

## 6. Crystallography.Controls の複合コントロール

`NumericBox` / `NumericBoxWithMenu` / `ColorControl` / `SizeControl` などは内部子コントロールが表面を覆うため、host の `ToolTip.SetToolTip` を本体だけに設定しても子の上で表示されないことがある。

共通方針:

- host フォーム側の標準 `ToolTip.SetToolTip(...)` を使う。
- 共有側の relay 実装がある場合は、内部子へ tooltip を配布し、二重 tooltip を抑止する。
- host provider に設定した `IsBalloon` / `AutoPopDelay` / `InitialDelay` / `ReshowDelay` が内部子にも伝わる設計を維持する。
- 共有コントロール独自の `ToolTip` プロパティを新規採用しない。必要な場合もデザイナ公開範囲を慎重に扱う。

共有コントロールの tooltip 挙動を変更した場合は、4 アプリの該当フォームで実機ホバー確認する。

## 7. 文案の adequacy 基準

良い tooltip は、ラベルの言い換えではなく、ユーザーが迷う点を補う。

- 何を入力・選択・実行するかが分かる。
- 単位、範囲、符号、結晶学上の前提が必要なら書く。
- ボタンは「押すと何が起こるか」を書く。
- チェックボックスは ON/OFF の差を明確にする。
- GUI ラベルと tooltip の用語をそろえる。
- 旧名、推測訳、別アプリ由来の表現を混ぜない。

## 8. 新設 provider チェックリスト

- [ ] `components` が無いフォームでは `System.ComponentModel.Container` を用意し、`Dispose` で破棄する。
- [ ] `toolTip = new System.Windows.Forms.ToolTip(components)` とする。
- [ ] `IsBalloon = true` を明示する。
- [ ] `AutoPopDelay = 10000` / `InitialDelay = 500` / `ReshowDelay = 100` を個別設定する。
- [ ] `AutomaticDelay` は触らない。
- [ ] resx に EN/JA tooltip 文を置く。
- [ ] 長文は 55 カラム目標で手動折返しする。
- [ ] 複合コントロールは host 側 `SetToolTip` で配線する。
- [ ] バルーン時は色・フォント指定を書かない。
- [ ] アプリ全体 ON/OFF がある場合は全 provider に効くか確認する。
- [ ] メニュー項目に tooltip を出すなら、対象 `MenuStrip` の `ShowItemToolTips = true` を設定する（§9）。
- [ ] 高 DPI、画面端、マルチモニタで表示位置を確認する。

## 9. メニュー / ToolStrip 項目のツールチップ

- `MenuStrip` は既定で `ShowItemToolTips = false`。そのため `ToolStripMenuItem` に `.ToolTipText`（resx + `ApplyResources`）を設定しても、メニュー項目では表示されない。メニュー項目に tooltip を出すには、対象 `MenuStrip` の `ShowItemToolTips = true` を設定する。
- `ToolStrip`（ツールバー）は既定で `ShowItemToolTips = true`。ボタンの `.ToolTipText` はそのまま表示される。
- トップレベルのメニューヘッダ（File / Edit 等）には付けず、ドロップダウン項目に絞る。
- `ToolStripItem` は `ToolTip` provider の対象外。文案は `item.ToolTipText`（`resources.ApplyResources(item, "item")` 経由）で与えるため、resx にキーを置くだけで適用される（per-item の `SetToolTip` は不要）。
